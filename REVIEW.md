# Adversarial Review — findings by component

Generated by running codex CLI against each major crate with an adversarial
prompt. Severity labels are mine. **Critical** = exploitable in the default
config. **High** = exploitable with minor preconditions, or guaranteed data
corruption/DoS. **Medium** = degrades correctness or observability but rarely
blocks a launch alone.

---

## crates/router (CRITICAL)

The biggest single source of risk. Almost every mutation and control-plane
route is unauthenticated.

- **Critical — session mint without credentials.** `POST /api/auth/session`
  accepts any `user_id` and returns a valid session.
  `crates/router/src/lib.rs:268,278`. Same for `POST /api/auth/upgrade`
  (lines `300,311`).
- **Critical — OAuth bypass.** `/api/auth/login/:provider` returns a fresh
  state to the caller; `/api/auth/callback/:provider` accepts a legacy
  `{email, state}` body without requiring an OAuth `code`. An attacker can
  request a state and mint a session as any email.
  `crates/router/src/lib.rs:447,452,455,469,481,493,535,544,563`.
- **Critical — entity mutations skip policies.** `GET /api/entity/...`
  enforces read policies, but `POST/PATCH/DELETE` bypass them entirely.
  `crates/router/src/lib.rs:1472,1489,1491,1495,1498`.
- **Critical — unauthenticated control plane.** Batch ops, sync push,
  transact, job enqueue/retry, scheduler trigger, and all workflow endpoints
  have no auth check at all.
  `crates/router/src/lib.rs:666,967,1345,1367,1546,1587,1654,1905,1957,1966,1981`.
- **High — identity spoofing in rooms.** Endpoints trust body-supplied
  `user_id` over `auth_ctx.user_id`, so an authenticated client can join/
  leave/set presence as another user.
  `crates/router/src/lib.rs:757,760,795,798,822,825`.
- **High — unbounded body parsing.** No size caps on `/api/import`, batch,
  transact, or sync push. Whole body is `serde_json::from_str`'d before
  validation. `crates/router/src/lib.rs:668,968,1236,1255,1346,1367`.

## crates/auth (HIGH)

- **Critical — `AuthContext` is public + `Deserialize`.** All privilege
  fields (`user_id`, `roles`, `is_admin`, `tenant_id`) are pub and directly
  trusted by `has_role()` and `AuthMode::check()`. If this type is ever
  deserialized from client-supplied JSON (which the router is currently
  positioned to do via body reads), it is straight privilege escalation.
  `crates/auth/src/lib.rs:11,15,16,19,89,149`.
- **High — guests satisfy "user" auth.** `AuthContext::guest()` sets
  `user_id: Some(...)`, and `AuthMode::User::check()` accepts anything with
  a user_id. Any route guarded only by `User` auth treats anonymous guests
  as authenticated. `crates/auth/src/lib.rs:48,84,149,947,1220`.
- **Medium — magic-code info leaks.** Error variants distinguish
  `NotFound`/`Expired`/`TooManyAttempts`/`BadCode`, enabling email
  enumeration. No IP/device rate limit.
  `crates/auth/src/lib.rs:636,643,646,676,717`.
- **Medium — session expiry off-by-one.** `is_expired()` uses `>` not `>=`,
  inconsistent with the rest of the codebase. `crates/auth/src/lib.rs:215`.

## crates/runtime/src/lib.rs (MEDIUM/HIGH)

- **High — FTS5 triggers are silently broken.** Trigger names are built as
  `"{quoted_fts}"_ai/_ad/_au`, which is invalid SQL; errors from the trigger
  `CREATE` are swallowed with `.ok()`. FTS indexes never sync on write.
  `crates/runtime/src/lib.rs:246,275`.
- **High — `query_graph` panics on invariant violation.** Uses `.unwrap()`
  on a lookup after an earlier check. `crates/runtime/src/lib.rs:865`.
- **Medium — fail-open query operators.** Unknown operators silently ignored;
  `$like` with non-string → `"%%"` (matches everything); empty `$in` → no
  predicate. `crates/runtime/src/lib.rs:749,758,777,1243,1262`.
- **Medium — read "pool" is deterministic sharding.** Round-robin picks one
  mutex with no fallback if it's poisoned/slow.
  `crates/runtime/src/lib.rs:1513`.
- **Medium — `query_graph` is non-atomic.** Multiple independent reads on
  potentially different pooled conns. Concurrent writes can produce
  inconsistent snapshots. `crates/runtime/src/lib.rs:861,875,880`.

## crates/policy (MEDIUM)

- **High — `hasAnyRole` delimiter injection.** `auth.hasAnyRole("billing,admin")`
  splits on comma and silently interprets as TWO roles, not one literal.
  Attacker-influenced role names change semantics.
  `crates/policy/src/lib.rs:203`.
- **Medium — extremely narrow operator set.** No `&&`, `||`, `!`,
  parentheses, null comparisons, field inequality, or tenant predicates.
  Schema validation only checks that `allow` is non-empty, so unsupported
  expressions are accepted at schema time and fail only at runtime.
  `crates/policy/src/lib.rs:149-268`, `crates/schema/src/lib.rs:318`.
- **Medium — denial messages leak raw policy expressions** into HTTP
  responses and WebSocket close reasons, giving attackers the full rule.
  `crates/policy/src/lib.rs:86,126,197,217`, `crates/router/src/lib.rs:1089`.

## crates/sync + crates/realtime + ws/sse (HIGH)

- **Critical — no auth or tenant scoping at the sync layer.** `PushRequest`
  and `ClientChange` let callers name arbitrary `entity`/`row_id`;
  `ChangeLog::pull` returns the global log. If the HTTP layer exposes these
  directly (it does — see router findings), any caller reads/writes anything.
  `crates/sync/src/lib.rs:71,77,117,137`.
- **High — replay/dedup protocol gap.** No request id / client id / nonce /
  expected base cursor. Retries become duplicate logical mutations.
  `crates/sync/src/lib.rs:71,117`.
- **High — change log eviction corrupts clients.** When old events are
  evicted, `pull` silently returns the surviving tail and advances the
  cursor instead of signaling "cursor too old / resync required". Clients
  converge to the wrong state. `crates/sync/src/lib.rs:137`.
- **High — unbounded broadcast channels + blocking writes.** Both WS and SSE
  hubs use unbounded `mpsc::channel`; shard workers do blocking
  `send`/`write_all`/`flush` while holding the shard mutex. One slow client
  stalls the hub and grows memory until OOM.
  `crates/runtime/src/ws.rs:42,86,131,221`, `crates/runtime/src/sse.rs:40,55,93,136`.
- **High — WS reader holds shard mutex across blocking `socket.read()`.**
  An idle client freezes the shard's broadcaster for the entire read
  timeout. `crates/runtime/src/ws.rs:221`.
- **High — no WebSocket frame/message limits.** `tungstenite::accept(stream)`
  with no `WebSocketConfig`, so a huge text frame is fully buffered before
  prefix checks. `crates/runtime/src/ws.rs:207,226,230`.
- **Medium — unauthenticated shard API.** `add_subscriber` and `push_input`
  are public and take a `SubscriberId` without verifying attachment. The
  `_authorized` variants are documented, not enforced.
  `crates/realtime/src/shard.rs:29,94,106,290,333`.

## crates/storage/src/sqlite.rs (MEDIUM)

- **No direct SQL injection found.** Identifiers go through `quote_ident`;
  values are parameterized.
- **High — schema DDL is not transactional.** `apply_schema_impl` runs each
  statement individually. Failure mid-migration leaves a partially-migrated
  schema. `apply_with_history` writes history after DDL in a separate
  step — schema can change without a history row.
  `crates/storage/src/sqlite.rs:198,217,341`.
- **Medium — identifier allowlist is missing.** `quote_ident` accepts any
  string. Pathological names are creatable but introspection excludes names
  starting with `_pylon_`, so such tables become invisible to planning.
  `crates/storage/src/sqlite.rs:52,82,435`.
- **Medium — corrupt history silently discarded** via `.ok()` on JSON
  parse. `crates/storage/src/sqlite.rs:268,306`.

## packages/functions/src/runtime.ts (HIGH)

- **High — `fn_type` enforcement is absent.** `handleCall` ignores inbound
  `msg.fn_type` and executes whatever is registered under `msg.fn_name`. A
  buggy/malicious peer can label a call as `query` while it actually runs a
  mutation, breaking host-side authorization assumptions.
  `packages/functions/src/runtime.ts:325,355`.
- **High — stdout corruption / JSON injection.** Protocol frames go to
  stdout. `console.log` is not fenced. Any handler or dependency that logs
  a JSON-looking line will be parsed as a protocol message by the Rust
  reader. `packages/functions/src/runtime.ts:57,241`.
- **High — `pendingRpcs` only allows one in-flight RPC per call_id.**
  `Promise.all([ctx.db.get(), ctx.runMutation()])` rejects the second call
  with an internal invariant error. If detached, that rejection is
  unhandled. `packages/functions/src/runtime.ts:66,128`.
- **High — protocol messages are blindly cast, not validated.** No
  structural check on inbound JSON beyond `type`.
  `packages/functions/src/runtime.ts:94,112`.
- **Medium — handler errors leak internals.** Raw `err.message` /
  `String(err)` returned to caller, including the runtime's own
  concurrency-invariant string.
  `packages/functions/src/runtime.ts:104,384`.
- **Medium — no RPC timeout / cleanup.** A dropped/truncated `result`
  frame leaves a promise pending forever until the parent kills the process.
  `packages/functions/src/runtime.ts:94,127`.

## crates/plugin/src/builtin (HIGH)

- **Critical — CSRF plugin is effectively a no-op.** `on_request()` always
  returns `Ok(())` because request headers are unavailable in that hook, and
  there's no synchronizer/double-submit token scheme — only origin matching.
  Unless every HTTP entrypoint manually calls `check()`, CSRF is not
  enforced. `crates/plugin/src/builtin/csrf.rs:36,67,102`.
- **High — password auth leaks user existence via timing.** Unknown emails
  return immediately; known emails pay the Argon2 cost. This is a
  user-enumeration oracle. Argon2 also uses `Argon2::default()` with no
  pinned work factor, so security is tied to crate version rather than an
  explicit policy choice. `crates/plugin/src/builtin/password_auth.rs:38,55,57,58,86,104,121`.
- **High — JWT refresh token replay cache is polluted before validation.**
  `refresh()` inserts the raw refresh string into `used_refresh_tokens`
  *before* cryptographic validation, so attacker-supplied garbage grows the
  set unbounded, and a seen-once real token is "burned" before expiry is
  checked. No `jti`/nonce, so two refresh tokens issued in the same second
  collide. `crates/plugin/src/builtin/jwt.rs:66,101,104`.
- **Medium — raw tokens printed via `Debug`.** `TokenPair`, `CreatedApiKey`,
  `TotpEnrollment`, `TrackedSession` all derive `Debug` with credential
  fields. Accidental `{:?}` logging dumps live secrets.
  `crates/plugin/src/builtin/jwt.rs:35`, `api_keys.rs:23`, `totp.rs:13`,
  `session_expiry.rs:8`.
- **Medium — TOTP is one-time flag, not MFA.** Success permanently sets
  `verified=true`; later checks only read that bool. No per-step counter to
  prevent replay within a 30 s window.
  `crates/plugin/src/builtin/totp.rs:63,67,77,131`.
- **Medium — api-key revocation race.** `resolve()` returns a detached
  `AuthContext`; `revoke()` only removes future lookups. In-flight/cached
  contexts survive revocation. `on_request` is a no-op.
  `crates/plugin/src/builtin/api_keys.rs:97,114,138`.
- **Medium — rate limiter collapses all anon traffic into one bucket.** No
  IP dimension despite the comment claiming per-IP. One attacker DoSes every
  anonymous user. `crates/plugin/src/builtin/rate_limit.rs:60`.
- **Medium — `session_expiry.refresh()` lets `expires_at` be renewed
  indefinitely**, defeating absolute-lifetime caps.
  `crates/plugin/src/builtin/session_expiry.rs:106`.

## crates/runtime/src/server.rs (HIGH)

- **Critical — missing auth before routing on fast paths.**
  `/api/files/upload`, shard SSE, fn streaming, and AI streaming all run
  BEFORE `pr.run_on_request()` and before `pylon_router::route()`, so any
  plugin-layer auth/policy middleware is skipped entirely.
  `crates/runtime/src/server.rs:446,563,580,727,815,944,991`.
- **High — body-size cap enforced after full read.** Both upload
  (`read_to_end`, 10 MB check after) and normal request (`read_to_string`
  before `MAX_BODY_SIZE` check) buffer everything first. Slowloris /
  memory amplification blocker. (Partially mitigated by my #102 fix;
  upload path still needs the same treatment.)
  `crates/runtime/src/server.rs:446,449,451,532,539,543`.
- **Medium — drain/shutdown sloppy.** Loop exits on any `recv()` error
  (not just intentional shutdown); `SHUTDOWN` never reset so in-process
  restart is broken; drain only waits for queued jobs, not in-flight
  HTTP/SSE or detached listener threads.
  `crates/runtime/src/server.rs:87,95,333,338,346,1027,1049`.
- **Medium — panic-prone `Header::from_bytes(...).unwrap()` on
  env-sourced CORS origin** at many sites.
  `crates/runtime/src/server.rs:291,388,999` and ~15 more.
- **Medium — `PYLON_CORS_ORIGIN` defaults to `*`** with `Authorization`
  allowed. In prod, any origin can drive bearer APIs.
  `crates/runtime/src/server.rs:291,711,800,925`.
- **Medium — connection exhaustion / starvation.** Main loop is synchronous;
  slow client or expensive handler blocks everyone behind. Streaming paths
  spawn unbounded OS threads.
  `crates/runtime/src/server.rs:333,446,532,685,761,891`.

## packages/react (HIGH)

- **High — WebSocket auth via URL query string.** `useShard` appends the
  bearer token to `?token=` in the WebSocket URL; it lands in proxy logs,
  browser devtools, and error telemetry. Only WS auth path offered.
  `packages/react/src/useShard.ts:29,107,111,112`.
- **High — `useShard` ignores `options` changes.** Token/subscriberId/
  baseUrl/reconnect-policy updates do not take effect until `shardId`
  changes. A logged-out user keeps the old socket under the old identity.
  `packages/react/src/useShard.ts:242,258,261`.
- **High — insecure default transport.** Both client and `db` silently fall
  back to `http://localhost:4321`. If the default ever escapes dev, session
  and function traffic ships in plaintext.
  `packages/react/src/index.ts:80,82`, `packages/react/src/db.ts:31,32,38`.
- **Medium — inconsistent token passing.** Low-level `useMutation` accepts
  `options.token`, but `db.useMutation`, `createTypedDb().useMutation`,
  `db.fn`, `db.streamFn`, and the upload wrappers all drop it.
  `packages/react/src/hooks.ts:289,291`, `db.ts:76,79,137,150,155`,
  `typed.ts:73,119,135`.
- **Medium — no `credentials: "include"` on fetches.** A remote `baseUrl`
  silently drops cookie-based auth.
  `packages/react/src/index.ts:93,170,193,227,346,381`.
- **Medium — `useMutation` / `useInfiniteQuery` set state after unmount.**
  No mounted guard; pending promises fire `setData/setError/setLoading`
  on dead components.
  `packages/react/src/hooks.ts:299,309,313,316,370-384`.
- **Medium — `useQuery.refetch()` is stale.** Mutates refs without
  triggering rerender; spinner/error state can stay invisible.
  `packages/react/src/hooks.ts:75,113,116,141,174,177`.

## packages/sync (HIGH)

- **Critical — offline mutations are silently lost.** `MutationQueue` is
  memory-only; never persisted to IndexedDB. Tab crash or refresh drops
  every pending write. Failed mutations are also immediately `clear()`ed,
  so permanent server rejects vanish from the UI.
  `packages/sync/src/index.ts:156,174,182,257`.
- **High — duplicate push writes under race.** `setInterval(push)` has no
  in-flight guard; a slow push overlaps the next tick and resends the
  same batch. Manual mutations also call `push()` synchronously, racing
  the timer. `packages/sync/src/index.ts:319,322,468,501,514,525`.
- **High — no idempotency key in protocol.** Client mints queue IDs but
  never sends them; server only receives raw `ClientChange`s. On retry
  after timeout there is no way to dedupe inserts. Optimistic inserts use
  `_pending_*` synthetic IDs with no reconciliation path.
  `packages/sync/src/index.ts:149,159,473,477,494`.
- **High — conflict resolution is "arrival-order merge".** `update`
  recreates missing rows; `delete` hard-removes them; no per-row version.
  Stale remote update after local delete **resurrects** the row; older
  update overwrites newer local state after reconnect.
  `packages/sync/src/index.ts:74,76,80,517`.
- **High — no tombstones.** Deletes wipe RAM + IndexedDB immediately; any
  replayed older insert/update recreates the row.
  `packages/sync/src/index.ts:80`, `persistence.ts:165`.
- **High — cursor can get ahead of persisted rows.** `applyChanges()`
  fires persistence asynchronously without awaiting; cursor persists
  after. A crash saves `last_seq` while some rows never reached disk,
  permanently missing those changes on restart. WS/SSE events update
  cursor in memory only.
  `packages/sync/src/index.ts:87,94,289,303,362,415`.
- **Medium — incoming `data.id` can override stored primary key** during
  apply/load, corrupting identity on reload.
  `packages/sync/src/index.ts:71,77`, `persistence.ts:101`.

## crates/cli (MEDIUM)

- **High — path traversal in `build` and `deploy`.** Route paths land in
  `staticgen::write_pages` which does `out_dir.join(page.path)` after
  only `trim_start_matches('/')`. Schema validation only checks "starts
  with `/`" and uniqueness. A route path like `/../../tmp/pwn` escapes
  the output directory. `crates/cli/src/commands/build.rs:56`,
  `crates/cli/src/commands/deploy.rs:276`, `crates/staticgen/src/lib.rs:52,68`,
  `crates/schema/src/lib.rs:391,401`.
- **High — Postgres DSN (with password) printed to stdout** by several
  schema subcommands, including when rendering success JSON.
  `crates/cli/src/commands/schema.rs:529,557,571,994`.
- **Medium — argv injection into Bun.** Paths are passed without a `--`
  separator; a filename starting with `-` would be interpreted as a flag.
  `crates/cli/src/bun.rs:9`, `crates/cli/src/commands/test.rs:72`.
- **Medium — Bun stderr copied into user-facing `Diagnostic.hint`** and
  printed verbatim. Secrets printed by the app or upstream stack traces
  land in the CLI output.
  `crates/cli/src/bun.rs:24`, `crates/cli/src/output.rs:44`,
  `crates/cli/src/commands/codegen.rs:55`,
  `crates/cli/src/commands/dev.rs:125`.
- **Medium — insecure deploy template.** Generated `docker-compose.yml`
  hardcodes `postgres://pylon:pylon@db:5432/pylon` and
  `POSTGRES_PASSWORD=pylon`.
  `crates/cli/src/commands/deploy.rs:144,154`.
- **Low — ad-hoc argv parsing.** Drops any token starting with `-`
  (no `--` end-of-options); `codegen client` dispatches on second
  positional so an entry file literally named `client` misroutes.
  `crates/cli/src/main.rs:38,47`,
  `crates/cli/src/commands/init.rs:25,33`.

## Biggest exploit chain

An unauthenticated caller can:
1. `POST /api/auth/session` → valid token for any `user_id`.
2. Or `POST /api/auth/callback/google` with `{email, state}` → session for
   any email.
3. With or without that session, directly call `POST /api/entity/...`
   (policies only guard reads) or `POST /api/sync/push` (no auth at all)
   to write anything to any entity.

## Recommended triage order

1. Gate every router mutation/control-plane route behind `AuthMode::User`
   or `AuthMode::Admin` and policy checks. **Blocker — fix first.**
2. Remove the legacy email/state OAuth callback path; require a real `code`
   exchange. **Blocker.**
3. Stop deriving `Deserialize` on `AuthContext` and strip `pub` from
   privilege fields. **Blocker.**
4. Enforce body size caps at the server layer (before `serde_json::from_str`).
5. Fix the sync protocol to carry idempotency keys + reject unknown
   cursors with "resync required".
6. Put `tungstenite::WebSocketConfig` limits on WS accepts; move WS reads
   off the shard mutex; make broadcast channels bounded.
7. Fix FTS5 trigger SQL (and stop swallowing DDL errors).
8. Wrap schema migrations in a single transaction with rollback on any
   failed step.
9. Fence TS runtime stdout (protocol to a dedicated FD or length-prefixed
   framing); validate inbound protocol frames structurally.
10. Enforce `fn_type` on the TS side; add RPC timeouts and orphan cleanup.
