# swift-todo

Minimal SwiftUI iOS/macOS app consuming the [`examples/todo-app`](../todo-app/) Pylon manifest. Demonstrates:

- `PylonClient` for HTTP (auth, file uploads, function calls)
- `PylonSync` for the local replica + WebSocket sync + offline mutation queue + SQLite persistence
- `PylonSwiftUI` for `ObservableObject` wrappers (`PylonQuery`, `PylonMutation`, `PylonSession`)
- Magic-code sign-in
- Optimistic insert / update / delete that survives app restart and network drops
- Generated typed Swift bindings via `pylon codegen client --target swift`

## Run

### 1. Start the Pylon server

```bash
cd examples/todo-app
pylon dev
# → http://localhost:4321
```

The dev server picks up the existing `pylon.manifest.json`; no separate setup needed.

### 2. (Re-)generate the Swift bindings

```bash
cd examples/swift-todo
pylon codegen client ../todo-app/pylon.manifest.json \
    --target swift \
    --out PylonTodoApp/PylonGenerated.swift
```

You only need to do this when the manifest changes. The pre-generated file ships with the example.

### 3. Build + run

**Open in Xcode:**

```bash
open Package.swift
```

Choose an iOS Simulator or "My Mac (Designed for iPad)" and hit ⌘R.

**Or via SPM CLI** (macOS only):

```bash
swift run PylonTodoApp
```

## What it does

1. **Sign-in screen** — enter an email, get a 6-digit dev code printed to the Pylon dev server's stdout, type it back, you're in.
2. **Todo list** — `PylonQuery<Todo>` keeps the list in sync. Mutations from other clients (open `localhost:4321/studio` and edit there) appear instantly.
3. **Add todo** — typed via the generated `client.createTodo(NewTodo(...))`. Optimistic — the row appears before the server confirms.
4. **Mark done / delete** — same shape; rolls back on server rejection.
5. **Offline support** — kill the network with airplane mode; mutations queue locally; reconnect drains the queue.
6. **Persistence** — kill the app entirely; restart; your data is still there because `SQLitePersistence` wrote it to `~/Documents/pylon-todo.db`.

## Architecture

```
PylonTodoApp/
├── App.swift              Entry point + DI for PylonClient + SyncEngine
├── ContentView.swift      Top-level routing (signed-in vs signed-out)
├── SignInView.swift       Magic-code sign-in flow
├── TodoListView.swift     PylonQuery<Todo>-backed list
├── PylonGenerated.swift   Generated by `pylon codegen client --target swift`
└── Info.plist             App config (App Transport Security exception for localhost dev)
```

## Switching to Pylon Cloud

Change one line in `App.swift`:

```swift
// let baseURL = URL(string: "http://localhost:4321")!
let baseURL = URL(string: "https://your-app.pylon.app")!
```

And remove the localhost App Transport Security exception in `Info.plist` since Cloud uses HTTPS.
